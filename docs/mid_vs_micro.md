# Mid vs Micro：中观因子加入的完整记录与对比

本文记录 5min 微观 CTA 流水线引入中观（mid-weekly）因子的全过程、已经实施过的两个版本（mid_v1 / mid_v2）的改动与结果，以及相对无中观基线（micro）的分品种对比与规律总结。

## 版本命名对照表

| 标签 | Run 目录 | 中观因子 | 三类 MID 特征（level / derived / AVAILABLE） | 其他关键配置 |
|---|---|---|---|---|
| **v0 · 仅微观** | `micro_result/` | 无 | — | 纯 5min 微观因子 + 工程特征 |
| **v1 · MID 含水平列** | `mid_v1_result/` | 有 | **level + derived + AVAILABLE 三类全入模** | `level_keep=true`（原始水平列保留）|
| **v2 · MID 去水平列** | `mid_v2_result/` | 有 | **derived + AVAILABLE 入模；level 全部 drop** | `level_keep=false`，其余同 v1 |

"水平列（level）"、"派生列（derived）"、"AVAILABLE 哑变量"三类特征的定义见 § 2.8。

对比报告文件：
- `results/comparison/midweekly_vs_baseline.md` — v1 vs v0
- `results/comparison/g1all_vs_baseline.md` — v2 vs v0
- `results/comparison/g1all_vs_candidate.md` — v2 vs v1

在 v1 和 v2 之间还做过一次不同方向的尝试：保留所有水平列，仅在源指标入模前加一个按频率归一化的稀疏度过滤（`min_active_ratio=0.6`）+ 识别并丢弃"阶梯哑变量"形态的指标。那次尝试的 run 目录已被清理，只保留了 A/B 结论——25 品种里只把原 v1 相对 v0 回归的 9 个品种修复了 5 个，剩 4 个仍退步，其中 1 个（RU）比 v1 更差（过滤误伤了它的关键列）。这条线索促成了后来把动作换成"直接 drop 水平列"，也就是 v2。

## 2. 加入中观因子的完整操作（pipeline 全景）

5min 训练矩阵接入 MID 因子经过 6 步处理，见 `pipeline/dataset.py::_merge_mid_weekly_features`（L653+）。本节描述 **v1 的操作**，v2 在此基础上只翻一个开关（§ 3）。

### 2.1 Step 1 — 读 xlsx 为稀疏观测表
`_read_mid_weekly_xlsx`：读 4 行表头，返回 `[timestamp, MID_<PID>_<ind_id>, ...]` 的宽表 + 元数据 `{indicator_id, indicator_name, frequency, ...}`。

### 2.2 Step 2 — 频率感知质量过滤（v1/v2 都开）
`_apply_mid_weekly_quality_filter`（L525+）：对每条源列：
- 若 `freq == 日`，`expected_ratio=1.0`；若 `freq == 周`，`expected_ratio=0.2`（business-day 网格）；
- `effective_ratio = nonnull_ratio / expected_ratio`；若 `< min_active_ratio=0.6` → drop；
- `drop_step_dummy=true`：首值位置 > 0 且首值后非空率 ≥ 0.9 → drop（识别"某日之前全空"的伪 regime 开关）。

被 drop 的列不产 level / derived / AVAILABLE。这条过滤在 v1 和 v2 之间不变。

### 2.3 Step 3 — 派生因子计算（在稀疏观测网格上）
`_compute_mid_weekly_derivatives`（L601+）：对每条 survive 的源列 `s = factor_df[col].dropna()`（只保留真观测的行）：
- 若 `len(s) < max(2, min(windows))=4` → 整条源列跳过派生；
- 对 `w ∈ [4, 13, 52]`（**观测数**，不是日历时间 → 周频列 4/13/52 ≈ 4/13/52 周；日频列 ≈ 4/13/52 交易日）：
  - `RET_{w}`：`s.pct_change(w).clip(-1.0, 5.0)`
  - `ZSCORE_{w}`：`(s − rolling_mean(w)) / rolling_std(w)`，`min_periods=max(1, w//2)`，std=0 置 NaN
  - `PCT_RANK_{w}`：`rolling(w).rank(pct=True)`，`min_periods=max(1, w//2)`
- 每源列**最多**产出 `3 windows × 3 transforms = 9` 条派生列。

### 2.4 Step 4 — 合并到 5min 网格
`pd.merge_asof(5min_bars, factor_df+derived_df, on=ts, direction="backward")` + 显式 `ffill()`（§ 10.1 不变量：禁止 `bfill` / forward-looking）。此步 level + derived 一起被铺到 5min 网格。

### 2.5 Step 5 — Staleness clamp
`ffill_max_bars=8064`（≈ 4 周 @ 5min）。每个文件维护一份 `__mid_arrival_ts__` 辅助列，累计 staleness > 8064 根 bar 时 level + derived 一起置 NaN。v1/v2 共用此阈值。

### 2.6 Step 6 — AVAILABLE 哑变量
`MID_<col>_AVAILABLE = merged[col].notna().astype(int8)`，**在 clamp 之后生成**，保证"观测已过期 → AVAILABLE=0"。受 `available_dummy=true` 控制。

### 2.7 Prefix 分流（`prepare`，L1050+）
- `MID_*` 列单独走 `mid_weekly.missing_ratio_relax=0.65`（默认 `max_factor_missing_ratio=0.35` 对周频列太严）。
- `MID_*` 的 NaN 在 dropna 前 fillna(0.0)，避免 dropna 杀行；`_AVAILABLE` 列同时记录"这个 0 是真缺失"。

### 2.8 每源列最多产出几列？

| 类别 | 命名 | 数量 | 开关 |
|---|---|---:|---|
| level | `MID_<PID>_<ind>` | 1 | `level_keep` |
| derived | `MID_..._{RET/ZSCORE/PCT_RANK}_{4/13/52}` | 9 | `derived.enabled` + `rolling_windows` + `transforms` |
| AVAILABLE | `MID_..._AVAILABLE` | 1 | `available_dummy` |

一源列最多 11 列；实际受 Step 2 过滤、Step 3 观测数不足跳过、Step 5 长期 stale、prepare 的 missing / 方差过滤约束。

## 3. mid_v1 与 mid_v2 的操作差异

### 3.1 配置差异（唯一一行）

| config 键 | v1 | v2 |
|---|---|---|
| `mid_weekly.level_keep` | `true`（保留原始水平列）| `false`（drop 全部水平列）|
| `mid_weekly.derived.enabled` | `true` | `true` |
| `mid_weekly.available_dummy` | `true` | `true` |
| `mid_weekly.ffill_max_bars` | `8064` | `8064` |
| `mid_weekly.min_active_ratio` | `0.6` | `0.6` |
| `mid_weekly.drop_step_dummy` | `true` | `true` |

**两个版本只差一行：`level_keep`。** 派生列、AVAILABLE 哑变量、staleness clamp、频率归一化过滤全部一致。

- v1：`level + derived + AVAILABLE` 全部入模。
- v2：`derived + AVAILABLE` 入模，**drop 全部原始水平列**。`_merge_mid_weekly_features` L747-750 `if level_keep ... else merged.drop(columns=level_cols_all)`.

### 3.3 为何只改这一处 —— 决策链回顾

- v1（含水平列）在 9 个品种上相对 v0 回归（FB/FU/Y/B/SN/RU/BB/JD/M，见 `docs/midweekly_regression_diagnosis.md § 14`）。
- 在 v1 和 v2 之间也尝试过另一种方向——保留全部水平列，仅在源头加频率归一化的稀疏度过滤。结果只把 9 个回归品种修复了 5 个（M, Y, B, JD, BB），剩 4 个仍退步；尤其 RU 甚至比 v1 更差（过滤误伤了它的关键列）。这说明"挑列过滤"不够，需要换动作。
- 根因分析：周频源列经 `merge_asof + ffill` 在 5min 网格上铺 ≈480 根 bar 的"分段常量"，LGB 把它当 regime flag 用，挤占骨干特征。**去掉水平列等于砍掉这个伪 regime flag 的载体**——这就是 v2 的唯一动作。

## 5. 分组特征分析：v2（去水平列）救了哪些品种、伤了哪些品种？

按 Δ(v2−v0) 分桶：

### 5.1 改善组（N=12，Δ ≥ +0.30）
`CU, EG, PB, SP, JM, BB, AG, JD, AU, BU, FB, RB`

### 5.2 持平组（N=9，|Δ| < 0.30）
`RR, B, ZN, PG, M, CS, V, Y, WR`

### 5.3 退步组（N=4，Δ ≤ −0.30）
`RU, SN, C, FU`

### 5.4 三组的 xlsx 组成中位数

| 组 | N | n_cols 中位 | **日频数** | **周频数** | **日频占比** | 日频 nonnull 中位 | 整体 nonnull |
|---|---:|---:|---:|---:|---:|---:|---:|
| 改善组 | 12 | 7 | **2.0** | **5.0** | **34.5%** | 0.688 | 0.364 |
| 持平组 | 9 | 7 | 4.0 | 4.0 | 50.0% | 0.578 | 0.327 |
| 退步组 | 4 | 8 | **4.5** | **3.5** | **59.2%** | 0.671 | 0.471 |

**关键规律 —— 日频列占比与 v2 效果强负相关**：
- **周频主导品种（daily_ratio < 50%）易被 v2 救**：改善组中位 daily_ratio 34.5%。典型代表 CU（日频 2/8 = 25%，Δ +1.72）、PB（2/7 = 29%，Δ +1.00）、BB（2/5 = 40%，Δ +0.76）、AU（1/6 = 17%，Δ +0.59）、FB（2/5 = 40%，Δ +0.39）。
- **日频主导品种（daily_ratio > 50%）易被 v2 伤**：退步组中位 daily_ratio 59.2%。典型代表 FU（5/8 = 63%，Δ −1.33）、C（7/12 = 58%，Δ −0.77）、SN（3/5 = 60%，Δ −0.65）。
- 持平组恰好在 50% 交界。

### 5.5 物理解释 —— 为什么日频 vs 周频决定 v2 效果

v2 的动作 = drop 所有水平列。水平列在 5min 网格上 = `merge_asof + ffill` 的稀疏观测值：
- **周频源列**：观测间隔 ≈ 480 根 5min bar（周一到周五每天 48 根，5 天 × 48 = 240 根工作日 bar；跨周末后 + 假期），ffill 段 = 480 根 bar 的"同一数字"。LGB 在这个 ffill 段上分裂 ≡ 把"最近一次观测的值高/低"当 regime flag，挤占 `CORR30 / STD60 / ATR` 等骨干 5min 因子的 gain。**v2 去掉这个伪信号载体 → 周频主导品种获益**。
- **日频源列**：观测间隔 ≈ 48 根 bar，ffill 段短 → 不会被 LGB 拿去当 regime flag；水平列本身是合理的价格/库存/现货副本，携带真信号。**v2 一起 drop 掉日频水平列 → 日频主导品种丢信号**。

这条规律在表里几乎逐行成立（除了 V、JM 这两个极端样本偏离）：
- **JM**（100% 日频，daily_ratio = 4/4）在改善组里**反常**：Δ +0.90。反例解释：JM 的日频列 nonnull 中位 0.38（相对较低），并非"干净的日频副本"；v2 把它们的水平列 drop 后 MID 列数从 43 降到 9，几乎等同于关掉了 JM 的 MID —— 结果反而好，说明对 JM 而言 MID **整体**是噪声。这是"`level_keep=false` 副作用等价于关 MID"的特例。
- **V**（17% 日频，周频主导）却在持平组边缘（Δ −0.24）：V 的唯一日频列 nonnull 0.536（中等），5 条周频列 nonnull 中位 0.249（较高）→ 其水平列在 v2 下被全部 drop，但剩下的派生列对 V 的骨干没补上来，属于"周频主导但 v2 没救成"的少数反例。
- **RU**（50% 日频）被划入退步组：位于规律边界，符合"持平组→退步组"的过渡段。

### 5.6 v2 vs v1 进一步验证规律

v2−v1 = "去掉水平列带来的增量"（以 v1 为基准）：
- v2 > v1（去水平列更好）：`FB(+3.02), BB(+0.99), JD(+0.81), JM(+0.71), B(+0.69), FU(+0.57), AU(+0.52), Y(+0.48), PB(+0.17)` —— **9/9 都是日频占比 ≤ 50% 的周频/混合品种**（例外：FU 日频 63%，但 FU 在 v1 下被 step-dummy 砸得太惨，v2 去掉水平列后反而救了一点；JM 日频 100% 属于"关 MID 也好"的 § 5.5 特例）。
- v2 < v1（去水平列更差）：`C(−1.97), WR(−1.22), RR(−0.69), PG(−0.59), BU(−0.36), CS(−0.38), AG(−0.27), V(−0.27), SN(−0.25), RU(−0.26), SP(−0.23), M(−0.02)` —— **多数是日频占比 ≥ 50% 或日频 nonnull 很高的品种**（C 58%、CS 67%、AG 18% 但日频 nonnull 0.97 极高、BU 38% 但日频 nonnull 0.987 极高）。

→ **结论**：v2（去水平列）作为全局规则，在"周频主导 + 日频少或覆盖低"的品种上是净收益；在"日频主导或日频 nonnull 极高"的品种上造成信号损失。

### 6.2 下一步可选路径（仍全局，不 per-pid）

- **候选 v3A（频率感知水平列）**：只对 `频率=周` 的源列 drop 水平列，`频率=日` 保留水平列。预期 C/SN/RU/FU 的日频水平列回归，改善组基本不动（因为它们本身日频少或本身就没从日频水平列获益）。
- **先做证据检查**：读 `mid_v1_result` 里 FU/C/SN/RU 的 feature_importance，看水平列 gain 是否真的排在高位。如果不是，说明 v2 伤这 4 个品种并非因为"丢了有信号的水平列"，而是间接原因（去掉水平列后派生列高度相关互相挤占、或水平 × 派生交互分裂缺失等），那 v3A 未必救得回来，需要换思路。
- **候选 v3B（收紧 `ffill_max_bars`）**：从 `8064`（≈4 周）降到 ≈ 96 根 bar（1 日），让 staleness clamp 更早把周频列置 NaN；不 drop 水平列，但让 ffill 段短，"分段常量"变"短段常量 + 更多 NaN 由 LGB 自己处理"。正交于 v2 的做法，可叠加。

按成本优先级，建议顺序：先做证据检查（几分钟，证伪假设）→ v3A / v3B（按证据结论选一或叠加）。
